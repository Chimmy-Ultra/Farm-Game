## 🚨 重大衝突（需 Kevin 裁決）

### 衝突 1：期貨系統定位

| 來源 | 決策 |
|---|---|
| **Notion 主頁** | 期貨複雜度 = **中度**（合約月 / 保證金 / 平倉 / P&L） |
| **這邊**（GAMEPLAY_DECISIONS） | **重新設計為「農民避險工具」，非交易模擬器** |

**衝突本質**：Notion 保留了完整期貨交易機制（可投機）；這邊限定只當避險用。

**我的建議（折衷）**：
- **機制層保留 Notion 版**（合約月、保證金、P&L），這是 29 商品 SSOT 已寫好的規格
- **框架層採這邊版**：UI tutorial 主打「農民避險」，分出兩個清楚的操作入口：
  - **避險模式**（主推）：「我種了 X 袋玉米，鎖定未來賣價」
  - **投機模式**（次要）：「不種也能買賣，風險自負」
- 新手被引導先用避險；進階玩家可解鎖投機

**這樣做的好處**：Notion 已寫的 commodities.ts 規格不用重做；這邊的「避險教學」定位也能實現；雙方成果都保留。

## 📢 我們這邊有、Notion 還沒記的決策

以下是 C/D/E 討論結論，需要告知另一邊（或我之後幫忙同步到 Notion）：

### A 區 — 角色與世界
- 固定等角視角（無旋轉，簡化實作）
- 上帝視角無角色
- 有體力系統（每日上限，睡覺回復）
- 混合式操作（點擊 + 拖曳）

### B 區 — 內容深度
- 工具升級線：**木 → 銅 → 鐵 → 金**
- 夜晚機制：**螢火蟲氛圍粒子 + 夜間作物（月光下成長）+ 老鼠害蟲**
- Stardew Valley 級別深度

### C 區 — NPC / 建築 / 教學
- NPC：**只做功能性**（像按鈕，無劇情/對話/好感度）
- 側活動：**完全不做**（無釣魚/採礦/節慶/戰鬥）
- 建築：**基礎 only**（穀倉、倉庫、圍欄），無裝潢
- 新手引導：**農會資料庫**（內建百科）+ Notion 的 2012 大旱 tutorial

### D 區 — 經濟進度
- 起始金幣：**200**（白手起家）
- 作物定價：**寫實比例**（玉米低利，咖啡/可可高利）
- 解鎖節奏：**成就制**
- 成就獎勵分層：金幣（初） / 解鎖新物品（中） / 裝飾徽章（後）

### E 區 — 體驗打磨
- UI 語言：**中英雙語**（i18n 從一開始做）
- 音樂音效：**延後**
- 部署：**GitHub Pages**
